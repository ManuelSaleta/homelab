Step 1: Deploy the Dashboard Workloads

Log into your k3s-control-01 node via SSH and run the official installation script. This pulls down all the required pods, replica sets, and core cluster architectures into a dedicated kubernetes-dashboard namespace:
Bash

kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml

(Note: Version 2.7.x is the stable production mainline for standard internal dashboard UIs. If you ever upgrade to the brand-new v3.x series down the road, it shifts over to a Helm-based paradigm).
Step 2: Create the Admin Service Account (The RBAC Pass)

By default, the dashboard is locked down tight with zero permissions. To log in and actually view your pods, namespaces, and system layouts, we need to create an administrative user.

Let's do this the clean, declarative way. Create a file named dashboard-admin.yaml:
YAML

apiVersion: v1
kind: ServiceAccount
metadata:
  name: admin-user
  namespace: kubernetes-dashboard
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: admin-user
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: admin-user
  namespace: kubernetes-dashboard

Apply it to the cluster:
Bash

kubectl apply -f dashboard-admin.yaml

Step 3: Expose It Natively via Your Traefik Ingress

Instead of dealing with manual, temporary kubectl proxy commands every time you want to check your pods, you can wire it straight into your new automated edge layout!

Create an Ingress manifest named dashboard-ingress.yaml:
YAML

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: kubernetes-dashboard-ingress
  namespace: kubernetes-dashboard
  annotations:
    ingressClassName: traefik
    # 🔐 CRUCIAL: The dashboard container speaks HTTPS internally. 
    # This tells Traefik to look past self-signed certs inside the cluster.
    traefik.ingress.kubernetes.io/router.serverstransports: "secure-transport@file"
spec:
  rules:
    - host: kube.freesalty.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: kubernetes-dashboard
                port:
                  number: 443

Apply it:
Bash

kubectl apply -f dashboard-ingress.yaml

Step 4: Mint Your Login Token

The dashboard will prompt you for a secure login token when you open it. Run this command to generate a dynamic Bearer Token for your newly minted admin user:
Bash

kubectl -n kubernetes-dashboard create token admin-user

Copy that long output string down to your clipboard.
Step 5: Route It to the World (Optional)

If you want to view this visualization when you're away from your desk, navigate back to your Cloudflare Tunnel dashboard:

    Click Add a route under your homelab-tunnel.

    Select Published application.

    Set Subdomain to kube.

    Set Domain to freesalty.com.

    Set Type to HTTPS (since the dashboard requires TLS internally).

    Set URL to 192.168.50.240 (Traefik).

Open up a browser tab, hit https://kube.freesalty.com, paste in your token, and enjoy your beautiful, realtime visual look into the heart of your Mothership cluster!
